# フロントエンドエンジニア コーディング課題

以下の要件に従って、TODOリストアプリケーションを作成してください。

## 1. アプリケーションの概要

基本的なCRUD機能を持つTODOリストを作成してください。また、**候補者が思う「Crazyな機能」を必ず1つ以上**盛り込んでください。

### データ構造

TODOアイテムは以下の要素を持つものとします。

| プロパティ名 | 説明 |
| :---- | :---- |
| id | TODOを一意に識別できる値 |
| title | TODOのタイトル |
| content | TODOの詳細内容 |
| completed | 完了状態 |
| createdAt | 作成日時 |
| updatedAt | 更新日時（作成時は作成日時と同じ） |

### 機能要件

* **CRUD操作**: 作成、読み取り、更新、削除が画面上で動作していること。  
* **データ永続化**: データベースの構築は不要です。  
  * APIレスポンスはMockで構いません。  
  * ただし、操作中にCRUDが機能していることがわかるようにしてください。  
  * また、リロードしたときにTODOが無くならないようにしてください。  
* **Crazyな機能**: 一般的なTODOアプリには存在しないような、独創的な機能を最低1つ実装してください。  
  * [イタンジのValues](https://corp.itandi.co.jp/about/mission)をもじったお題であり、機能のCrazyさ（奇抜さ）は評価対象ではありません。  
  * あくまで技術力を見るための課題ですので、気負わず作成してください。

## 2. 技術要件

### 必須フレームワーク・パッケージ

環境構築時点での最新安定版を使用してください。

* **Next.js** (※ **Pages Router** を使用すること)  
  * [create-next-app](https://nextjs.org/docs/app/api-reference/cli/create-next-app) を使用してください。  
* **TypeScript**  
* **React**

### スタイリング

CSSの基礎実装力を確認したいため、以下の制約を設けます。

* **NG**: Tailwind CSSなどのUtility Firstフレームワーク、Bootstrap、MUIなどのUIコンポーネントライブラリの使用は不可とします。  
* **OK**: フレームワークやUIコンポーネントライブラリに依存しない手法（CSS Modules, Emotion, styled-componentsなど）を使用してください。  
* **補足**: デザインの美しさは評価対象ではありませんが、レイアウト崩れがなく、操作可能な状態にしてください。

## 3. 生成AIの利用について

生成AI（ChatGPT, GitHub Copilot等）の使用は 許可 します。  
ただし、後述するREADME.mdにて、どの部分を自身で実装し、どの部分をAIに任せたか（またはAIの支援を受けたか）を明確に記載してください。

## 4. 提出物・ドキュメント

ソースコードのリポジトリ（GitHub等）を共有いただくか、Zipファイル等で提出してください。  
ルートディレクトリに README.md を配置し、以下の内容を記載してください。

1. **Crazyな機能の説明**:  
   * どのような機能を実装したか、その意図やアピールポイント。  
2. **動作確認方法**:  
   * アプリを起動し、CRUD操作を確認するまでの手順。  
   * Docker等を使用した場合は、その起動方法も含めてください。  
3. **技術選定の理由**:  
   * create-next-app によって追加されるパッケージ以外にパッケージを追加した場合や、設定（next.config.jsやeslint.config.jsなど）を変更した場合は、その理由を記述してください。  
4. **AI活用の範囲**:  
   * 実装におけるAIの関与度合いについて。  
5. **作業時間**:  
   * 1営業日を8時間として完成までに何営業日かかったか。
